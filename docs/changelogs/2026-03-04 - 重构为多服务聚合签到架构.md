# 2026-03-04 重构为多服务聚合签到架构

## 概述

将项目从单脚本（`anyrouter_checkin.py`）重构为可扩展的多服务、多账号聚合签到框架，并新增 Facai 签到服务。

## 变动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| **新增** | `config.py` | 统一配置加载，解析 `#` 分隔的多账号环境变量，支持 `extra_fields` 扩展字段 |
| **新增** | `checkin/__init__.py` | 签到服务注册表 `CHECKIN_REGISTRY` + `@register` 装饰器 |
| **新增** | `checkin/base.py` | 签到抽象基类 `BaseCheckin`，提供 `run()` 和 `run_all()` |
| **新增** | `checkin/anyrouter.py` | AnyRouter 签到服务（从旧脚本迁移） |
| **新增** | `checkin/facai.py` | Facai (New-API) 签到服务，支持 `New-Api-User` header 认证 |
| **新增** | `main.py` | 统一入口，支持 `--service` 参数筛选服务 |
| **新增** | `.env.sample` | 环境变量示例（ACCOUNT / COOKIE / USERID） |
| **新增** | `AGENT.md` | AI 协作规范 |
| **新增** | `pyproject.toml` | uv 项目配置 |
| **新增** | `uv.lock` | uv 锁文件 |
| **重命名** | `.github/workflows/anyrouter_checkin.yml` → `daily_checkin.yml` | 统一工作流，注入所有服务的 secrets |
| **修改** | `.gitignore` | 添加 `__pycache__/` |
| **修改** | `requirements.txt` | 添加 `python-dotenv`、`httpx` |
| **删除** | `anyrouter_checkin.py` | 逻辑已迁移至 `checkin/anyrouter.py` |

## 架构设计

```
main.py (入口)
  ├── config.py (加载 .env，解析多账号)
  └── checkin/ (签到服务包)
        ├── __init__.py (@register 注册表)
        ├── base.py (BaseCheckin 抽象基类)
        ├── anyrouter.py (Playwright + WAF 绕过)
        └── facai.py (Playwright + New-Api-User header)
```

### 添加新服务流程

1. `checkin/` 下新建模块 → 继承 `BaseCheckin` → 加 `@register`
2. `main.py` 中 `import checkin.xxx`
3. `.env.sample` + CI workflow 添加对应环境变量
